# Phase 7 希望入力・結果確定運用

## 希望入力

1. 席配置を公開すると、同じテーブルの参加者が会話相手として登録される。
2. イベント設定で希望方式と受付期間を確認する。
3. 管理画面から希望入力を開始する。
4. 参加者は会話済み・受付済み・回避対象外の相手だけを選択できる。0名提出も可能。
5. 締切後、管理画面から希望入力を締め切る。

一方希望、順位、任意メモは相手へ表示しない。参加者APIは本人が保存した希望と、責任者確定後の成立結果だけを返す。

## 結果確定・通知

1. 双方希望候補を `approved` / `declined` / `pending` に分類する。
2. 複数成立不可の場合は競合をすべて解消する。
3. `pending` と未処理候補が0件であることを確認する。
4. manager が対象人数と成立組数を確認して結果を確定する。
5. 通知プレビューAPIで通知対象、成立参加者、不成立参加者、成立組数を確認する。
6. 同じ件数を送信APIへ渡して再確認し、通知を予約する。

確定前の結果は参加者APIへ返らない。確定取消は system_admin のみ実行でき、理由が必須。LINE送信失敗は既存の通知再送ジョブで処理する。
